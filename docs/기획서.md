## 0. 한 줄 정의

최애 멤버 1명이 파우더블루 픽셀 세계의 배경 21곳에서 **살아 움직이는 다마고치형 인터랙티브 홈** 웹앱. 매일 들러 "물방울"을 모으고, 새 배경을 해금할 때마다 멤버의 새로운 연출을 만난다.

- **앱명 후보**: 룸:블루(ROOM:BLUE) *(추천)* / 여름한켠 / 포켓 썸머룸 — 그룹명·실명 미사용
- **타겟**: TWS 팬덤 (모바일 세로, 브라우저에서 바로 열림)
- **목표 감정**: "내 폰 속에서 나의 소년이 여름을 살고 있다"
- **톤**: 저채도 파우더블루·크림, 청량·청순 여름, 픽셀아트 (`assets/generated/asset-plan.json`의 styleLock 준수)
- **형태**: 정적 웹앱, 서버 없음(localStorage), GitHub Pages 배포

## 1. 절대 제약

1. `assets/generated/` 아래 **이미 존재하는 이미지만** 사용한다. 새 이미지 생성 금지. (CSS/코드로 그리는 UI, 기존 이미지의 리사이즈·포맷 변환은 허용)

3. 비공식·비상업. 광고/과금 없음.

```

      │                                        │
```

**재화 수치 설계**
- 출석 +30 / 멤버 탭 +1(일 20회 캡) / 7일 연속 출석 보너스 +100 → 일평균 ~50물방울

- 전체 해금 총액 ≈ 3,200물방울 ≈ **6~8주 플레이**
- **scene-21은 재화로 못 삼** — 20종 전체 수집 시 자동 해금되는 엔드콘텐츠 보상

## 3. 핵심 설계 결정: 2-레이어 캐릭터

## 6. 인터랙션 설계

- **탭(멤버)**: 머리 위 CSS 픽셀 말풍선에 `expression-laugh/shy/surprised` 배경별 가중 랜덤 1.5초 + CSS 하트 파티클 + 물방울 +1(일 20캡). 연속 5탭 = "간지럼"(surprised→laugh 콤보)
- **더블탭**: 배경별 보조 포즈(§5 "탭 반응 특이" 열)
- **방치 90초**: 수면 모드 — 딤 60% + sit 포즈 + CSS "Zzz". scene-02에서는 sleeping 컷 비네트 추가. 탭 해제 시 surprised
- **무드컷 비네트 탭**: 확대 뷰, 2컷 이상 씬은 arrow로 넘기기
- **스와이프 좌/우**: 해금 배경 간 이동(크로스페이드 300ms, 다음 배경 프리로드 완료 시에만 전환)

```json

{
  "member": "C3",
  "names": {"C1": "멤버 1", "C2": "멤버 2", "C3": "멤버 3", "C4": "멤버 4", "C5": "멤버 5", "C6": "멤버 6"},
  "droplets": 240,
  "lastVisit": "2026-07-02",
  "streak": 4,
  "tapEarnedToday": 12,
  "unlocked": ["scene-01", "scene-02", "scene-07"],
  "currentScene": "scene-07",
  "umbrellaColor": "mint",
  "flags": {"seen21Credit": false},
  "settings": {"reducedMotion": null}
}
```

- 출석: 접속 시 KST 날짜 비교(연속=streak+1, 끊김=1 리셋). `lastVisit > today`(시계 되돌림)면 그날 지급만 스킵
- 상태 변경 시 debounce 500ms 일괄 기록. 스키마 버전 `v1`로 마이그레이션 대비
- 설정에서 JSON 텍스트 내보내기/가져오기 → 기기 변경 대응

## 8. Phase 2 / Phase 3

**P2 픽셀 포카 메이커** (진입: 홈 HUD 카메라 버튼)
- 캔버스 941×1672 고정. 조합: 프레임 3종(`frame-camera` 게임기 / `frame-summer-patchwork` / `frame-wavy-yellow`) × 배경(해금분) × 멤버 포즈 14종(드래그·스케일) × 표정 스탬프 4종 × 이펙트 3종 × 픽셀 웹폰트 텍스트(예: Galmuri)

**P3 우산 미니게임 "빗방울 받기"** (진입: scene-18 룸)
- 배경 `scene-18-bg-topview`, 플레이어 `C{n}-umbrella-topview` 드래그/터치 추적, 우산 6색 스킨
- `effect-water-droplets` 프레임 기반 빗방울 낙하 60초(점점 빠름), 우산 반경 판정 → 콤보
- 점수÷10 물방울 지급(일 상한 30). requestAnimationFrame + 단일 canvas

## 9. 기술 스택 & 성능 전략

- **Vite + TypeScript 바닐라 (프레임워크 없음)**. 화면 3~4개·단순 상태라 React 과함. 뷰 전환은 상태 기반 단일 페이지
- **스프라이트 애니**: walk/run 시트 = `width:181px; animation: walk .8s steps(8) infinite` CSS steps(합성만 발생, 60fps 안전). water-droplets 동일. 포즈 전환 = `<img>` src 스왑 + 크로스페이드. 전 이미지 `image-rendering: pixelated`
- **43MB 배경 대응 (핵심)**: 빌드 스크립트(`sharp`)로 파생본 생성 —
  ① WebP 변환 + 최대변 1280px 리사이즈(장당 목표 150~300KB, 픽셀아트는 lossless 옵션도 비교 검증)
  ② 컬렉션용 256px 썸네일
  ③ 20px 블러 LQIP
  런타임: 현재 배경 + 좌우 인접 1장만 프리로드. **원본 PNG 무수정**, 파생본만 `app/public/opt/` 출력
- 캐릭터/이펙트/UI ~10MB는 **선택 멤버 것만 로드**(멤버당 ~1.5MB)
- **GitHub Pages**: `base: '/pledis_app/'`, Actions로 `app/dist` 배포. (MVP 이후) 서비스워커로 2회차 방문부터 오프라인 다마고치
- 검증 게이트: 4배속 CPU 스로틀 60fps, LCP < 2.5s

## 10. 안전 설계 (비공식 팬앱)

- 내부 식별자·에셋·세이브 전부 **C1~C6 슬롯만**. 실명은 코드/레포 어디에도 없음. 표시명은 사용자가 직접 입력(localStorage에만 존재) → 배포물에 실명 제로
- 앱명·도메인에 그룹명 미사용, 스플래시/설정에 "본 앱은 비공식 팬메이드이며 특정 소속사와 무관합니다" 상시 고지, 문의 시 게시 중단 안내
- 광고·과금 없음, 포카 저장물 FANMADE 워터마크

## 11. MVP 범위와 구현 순서

**MVP(Phase 1)**: 온보딩/멤버 선택, 홈 룸(우선 배경 8종: 01~04, 07, 11, 17, 18), 탭 표정 반응, 수면 모드, 출석/재화/해금, 컬렉션 도감, 설정. — 제외: 포카 메이커, 미니게임, 무드컷 확대 뷰(토스트 대체)

1. `tools/optimize-assets.mjs` — sharp 파이프라인(WebP/썸네일/LQIP) + 매니페스트 JSON
2. `app/src/data/scenes.ts` — §5 매핑 표를 데이터로 코드화
3. 코어: 스토어(localStorage)+출석 → 홈 렌더러(레이어 4단)+스프라이트 엔진(CSS steps)
4. 인터랙션(탭/더블탭/방치/스와이프) → 컬렉션/해금 → 온보딩 → 설정
5. 저사양 검증 후 나머지 13개 배경 연출 데이터 추가
6. Phase 2 포카 메이커 → Phase 3 미니게임
