## 0. 한 줄 정의

TWS 뮤비 「첫 만남은 계획대로 되지 않아」를 **픽셀아트 컷씬**으로 재구성한, 화면에 한 장면씩 크게 보여주고 `< >`로 넘기는 **슬라이드 + 컷씬 재생기** 웹앱.

- **타겟**: TWS 팬덤
- **목표 감정**: "원본 뮤비가 바로 떠오른다" + "와 이것까지 구현했어?"
- **톤**: 청춘·청량, 픽셀아트
- **핵심 서사**: 혼자 계획 세우던 1명 → 하루 동안 예상 밖의 사람들을 만남 → 마지막엔 다 함께. **화면 속 인원이 점점 늘어나는 곡선**이 또렷해야 한다.
- **정체성**: 완벽한 아이돌 안무가 아니라 **멈칫·헛발질·놀람·민망한 웃음** 같은 어설픔. 디테일 안무보다 **대형 변화**.

## 1. 아키텍처 결정 (team-lead 최종 / ADR)

**Canvas2D 단일 렌더러 + GSAP 값 구동 + DOM은 UI 한정.**

| 레이어 | 담당 | 내용 |
|---|---|---|
| **Canvas2D (단일)** | 렌더 전부 | 배경, 6캐릭터 스프라이트, 파티클(비·흩어짐·낙하), 씬 A→B 전환 비주얼, 드래그 물리(농구공 등) |
| **GSAP (no-DOM)** | 모션 구동 | progress/좌표/카메라 값 트윈 + timeline 시퀀싱. **DOM 트랜스폼 트윈 안 함** |
| **DOM/HTML** | UI만 | `< >` 버튼, 진행 인디케이터, 자막, 메뉴. 씬 재생 중 `backdrop-filter` 0개 |
| **Swiper** | **제외** | 스와이프는 Canvas 위 포인터 이벤트 → GSAP 씬 전환 직접 호출. 슬라이드 라이브러리 추가 안 함 |

**근거**: 픽셀 정합(integer scaling), `imageSmoothingEnabled=false` 통제, 씬17 비 200개·군중 대형 변화에서 모바일 60fps를 지키려면 렌더 경로가 하나여야 한다. GSAP은 DOM을 움직이는 도구가 아니라 **Canvas 씬의 값(progress 0→1)을 구동하는 타임라인 엔진**으로 쓴다.

## 4. 모션 사전 (motion-engineer) — *GSAP가 Canvas 값을 구동*

| 동사 | 모션 | 적용 씬 | duration | ease | reduced-motion 폴백 |
|---|---|---|---|---|---|
| 낙하(fall) | 중력 가속 이탈→위에서 진입 | 03,12,16 | 500ms | in:`power2.in`/out:`power2.out` | 즉시 컷 |
| 확대축소(zoom) | scale 정수배 이탈/진입 | 02,05,13,18,20 | 650ms | `power3.inOut` | 즉시 컷 |
| 흩어짐(scatter) | 조각 stagger 방사(≤12조각) | 10,14 | 400+stagger25ms | `power3.out` | 즉시 컷 |
| 슬라이드(slide) | translate 이탈/진입 | 04,06,08,11,15,19 | 350–550ms(08:250) | `power2.inOut` | 즉시 컷 |
| 등장(enter) | 요소별 진입(문/바운스/빗줄기) | 07,09,17 | 550–700ms | 09:`back.out(1.4)` 등 | 즉시 노출 |

- **슬랩스틱 캐릭터 애니**: 스프라이트 시트 프레임(기본 8fps, 절정 12fps). 좌표 보간 X(픽셀 보존).
- **공통 전환 셸**: 키보드 ←→ / 버튼 / 스와이프 → `triggerTransition(dir)` 하나로 수렴. `SCENE_TRANSITION_MAP` 룩업으로 씬별 타입 분기. `isTransitioning` 락.
- **reduced-motion 전역**: `gsap.defaults({duration: REDUCED ? 0 : undefined})` 한 곳에서 OFF. 콜백·서사 흐름 유지. 씬12 드래그는 "다음 →" 버튼으로 대체, 씬18 탭은 ✓ 즉시 표시.
- **픽셀 주의 3**: ① 모든 translate `Math.round()`(modifiers) ② scale 정수배 + `image-rendering:pixelated` 고정 ③ `filter:blur`·모션블러 트윈 금지, `will-change`는 전환 직전 부여·`onComplete`에서 제거.

## 5. 성능 가드레일 (perf-engineer) — *타협 불가 한계선*

| 항목 | 한계선 |
|---|---|
| 배경 1장 | ≤35KB (인덱스 PNG-8, 16색) |

| 군중(씬14) | 개별 스프라이트 **0개** — 배경에 굽기 + 2프레임 흔들림 타일 |
| backdrop-filter | 씬 재생 중 **0개** (남색 반투명 단색으로 대체) |
| 모바일 목표 | 60fps (50fps 이하 시 파티클 자동 감축) |
| Canvas 내부 해상도 | 360×640 고정 (DPR 곱 금지) |

**에셋 파이프라인**: AI 원본(1024²) → ① nearest-neighbor 다운스케일(Lanczos 금지) → ② pngquant 16색 양자화(마스터 팔레트 고정) → ③ oxipng 무손실 → ④ 상한 초과 시 팔레트/해상도 감축 → ⑤ `pixelated` 상태로 브라우저 검증.

**로딩**: 씬 N±1 항상 상주 + N±2 프리로드(`requestIdleCallback`), N-2 이하 해제. 씬01 BG는 `<link rel=preload as=image fetchpriority=high>`. Canvas `aspect-ratio:9/16`로 CLS 방지.
