## 0. 한 줄 정의

최애 멤버 1명이 파우더블루 픽셀 세계의 21개 공간에서 **살아 움직이는 다마고치형 인터랙티브 홈** 웹앱. 매일 들러 "물방울"을 모으고, 새 공간을 해금할 때마다 멤버의 새로운 하루 장면을 만난다.

- **앱명**: 룸:블루(ROOM:BLUE) *(확정 후보 1순위)* / 대안: 여름한켠, 포켓 썸머룸
- **타겟**: TWS 팬덤. 모바일 세로 웹(브라우저에서 링크로 바로 열림)
- **목표 감정**: "내 폰 속에서 나의 소년이 여름을 살고 있다"
- **형태**: 정적 웹앱, 서버 없음(localStorage), GitHub Pages 배포
- **세계관 카피**: "밤의 책상에서 시작해, 콘페티가 내리는 옥상에서 끝나는 어느 여름의 하루" — 21씬이 실제로 밤 방(01)→아침(03)→등교(07~09)→낮(10~15)→비(16~18)→노을(19~20)→피날레(21)의 하루 곡선을 이룬다. 씬 해금 순서가 곧 서사다.

## 1. 절대 제약

1. `assets/generated/`의 **기존 이미지만** 사용. 새 이미지 생성 금지(코드 드로잉 UI·리사이즈·포맷 변환·투명화 보정은 허용).
2. 실명 금지 — 코드·에셋·세이브 전부 C1~C6 슬롯(§10).
3. 비공식·비상업. 광고/과금 없음.
4. generic-* 캐릭터 에셋은 사용 금지(프레임 폭 비정수 — spec-assets §2).

## 5. 멤버 행동 상태 머신 (홈 공통)

```
        ┌──────────────────────────────┐
        ▼                              │
  ENTER → IDLE_A ──(씬별 주기)──> IDLE_B ─┘
   │        │ tap                │ tap
   │        ▼                    ▼
   │      REACT(표정 말풍선 1.5s) ──> IDLE_A
   │        │ 5연타 이내
   │        ▼
   │      TICKLE(surprised→laugh 콤보 2.5s) → IDLE_A
   │        │ dbltap
   │        ▼
   │      SPECIAL(씬별 액션) → IDLE_A
   └─ 무입력 90s → SLEEP(딤 60% + sit + Zzz) ─ tap → WAKE(surprised 2s) → IDLE_A
```
- REACT 중 추가 탭은 큐잉하지 않고 카운터만 증가(5연타 판정용). SPECIAL 중 입력 무시(최대 3s).
- SLEEP 중에는 물방울 적립 없음. 씬 전환·화면 이탈 시 상태 리셋 → ENTER.
- `document.hidden` 시 모든 rAF·타이머 정지(배터리), 복귀 시 ENTER부터.

## 6. 시스템 설계

### 6-1. 세이브 (localStorage `roomblue.save.v1`)
```jsonc
{
  "v": 1,
  "member": "C3",
  "names": {"C1":"멤버 1","C2":"멤버 2","C3":"멤버 3","C4":"멤버 4","C5":"멤버 5","C6":"멤버 6"},
  "droplets": 240,
  "lastVisit": "2026-07-02",     // KST(Asia/Seoul) 날짜 문자열
  "streak": 4,
  "tapEarnedToday": 12,
  "unlocked": ["scene-01","scene-02"],
  "currentScene": "scene-02",
  "umbrellaColor": "blue",
  "flags": {"seen21Credit": false, "firstTapToast20": false},
  "settings": {"reducedMotion": null}   // null=시스템 따름
}
```
- 기록: 상태 변경 debounce 500ms + `visibilitychange`(hidden) 시 즉시 flush.
- 마이그레이션: 로드 시 `v` 검사 → 버전별 마이그레이터 체인. 파싱 실패 시 백업 키(`.bak`)로 이동 후 새 세이브(조용히 초기화하지 않음 — 복구 안내 노출).
- 출석 판정: `Intl.DateTimeFormat('sv-SE',{timeZone:'Asia/Seoul'})`로 오늘 날짜 산출. 어제=연속, 그 외 과거=streak 1 리셋, **미래(시계 되돌림)=지급만 스킵**. 접속 중 자정 통과는 다음 상호작용 시점에 재판정.

### 6-2. 엣지 케이스
| 상황 | 처리 |
|---|---|
| localStorage 사용 불가(프라이빗 모드 등) | 인메모리 폴백 + 상단 경고 배너 "이 브라우저에선 진행이 저장되지 않아요" |
| 탭 2개 동시 오픈 | `storage` 이벤트 수신 → 최신 세이브로 소프트 리로드 |
| 배경 로드 실패 | LQIP 유지 + 3회 지수 백오프 재시도 + 수동 재시도 버튼 |
| 세이브 가져오기 검증 실패 | 스키마 검증(필드·범위) 후 거부, 기존 세이브 불변 |
| 잔액 부족 해금 시도 | 버튼 비활성 + "💧 60 더 필요해요" |
| 구형 브라우저(WebP 미지원) | `<picture>` PNG 원본 폴백(용량 경고 토스트) |

### 6-3. 접근성
- 터치 타깃 최소 44×44px. HUD 아이콘에 aria-label. 도감 카드 = 버튼 시맨틱.
- `prefers-reduced-motion` 기본 존중: 팬·루프·파티클 정지, 포즈는 정지 프레임, 크로스페이드는 유지(150ms).
- 말풍선·토스트는 텍스트 병기(표정 이미지에 alt 상당 정보). 명도 대비 4.5:1(HUD 텍스트는 반투명 스크림 위에).
- 사운드 없음(BGM/효과음 미도입 — 저작권·용량·자동재생 정책 회피). 진동은 시그니처 액션에만 `navigator.vibrate(30)` (지원 기기 한정).

## 7. Phase 2 — 픽셀 포카 메이커 (요약 스펙)

- 진입: 홈 📷. 캔버스 941×1672 고정(프레임 원치수), 미리보기는 fit.
- 레이어: 배경(해금분만) → 멤버 포즈(14종, 드래그·핀치 스케일 0.5~2.0×·좌우 반전) → 표정 스탬프(4종, 최대 3개) → 이펙트 스탬프(구름/불꽃/물방울 프레임) → 프레임 3종 → 텍스트 1줄(픽셀 웹폰트 Galmuri9, 색 6종).
- 출력: Canvas 2D `imageSmoothingEnabled=false` → `toBlob('image/png')` 저장 + Web Share API(파일 공유 지원 시). 우하단 "FANMADE" 워터마크 자동(프레임 위, 제거 불가).
- 재화: 제작 완료 시 10 소모(미리보기는 무료). 일 제작 수 제한 없음.

## 8. Phase 3 — 우산 미니게임 "빗방울 받기" (요약 스펙)

- 진입: scene-18 더블탭. 단일 canvas + rAF.
- 플레이: `C{n}-umbrella-topview`(+선택 우산색)를 터치 추적 이동(lerp 0.18). 상단에서 droplet-f1~f3 낙하(초속 140→60s에 320px/s 선형 가속, 스폰 간격 900→350ms). 우산 캐노피 반경 판정 → f5 스플래시 + 콤보(0.8s 유지). 웅덩이 좌표는 감점 아님(장식 물결).
- 종료: 60s. 점수 = 받은 방울×(1+콤보/10). 지급 = 점수÷10(일 30 캡, 캡 도달 시 "내일 또!").

## 9. 기술 스택 & 성능

- **Vite + TypeScript 바닐라**(프레임워크 없음): 화면 4개·단일 스토어에 React는 과함. 상태 기반 단일 페이지, 뷰는 DOM 교체.
- 스프라이트: walk/run/droplet = CSS `steps()` 배경시트 애니(컴포지터 전용). 포즈 전환 = `<img>` src 스왑+크로스페이드 120ms. 전 이미지 `image-rendering: pixelated`.
- 이미지: sharp 파이프라인(spec-assets §7) — 배경 43MB→~5MB WebP, 캐릭터는 lossless. 초기 전송 <800KB, LCP <2.5s(4G), 4×CPU 스로틀 60fps 게이트.
- 배포: GitHub Pages, `base:'/pledis_app/'`, Actions로 `app/dist`. 서비스워커는 MVP 이후.
