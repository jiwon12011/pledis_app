# 룸:블루 — 21씬 연출 상세 스펙

> `docs/기획서.md`의 부속 문서 B. 배경 구도·좌표는 원본 이미지 육안 분석 실측(2026-07-02).
> 이 문서가 `app/src/data/scenes.ts`의 원본이다 — 씬 하나 = 데이터 블록 하나로 1:1 코드화한다.

## 0. 공통 규칙

**좌표계**: 배경 원본 픽셀 기준 %, (x: 좌→우, y: 상→하). 멤버 위치는 **발끝(또는 앉은 엉덩이) 기준점**.
**멤버 표시 크기**: `scale`(씬별 명시) = 멤버 신장 ÷ 화면 세로높이. 근경 0.30~0.34, 중경 0.24~0.28, 탑뷰는 이동영역 대비 0.16~0.20.
**무드컷 비네트**: `cutscene-frame-small` 안에 컷을 82% 크기로 삽입. 기본 위치 = 우상단(x82%, y14%), 렌더 폭 = 화면의 26%. 탭 → 확대 모달(화면 84% 폭), 컷 2장 이상이면 arrow-left/right로 순환. 등장: 씬 진입 3초 후 fade+scale in.
**행동 타임라인 표기**: `IDLE_A`(기본 루프) / `IDLE_B`(간헐 연출, 주기 명시) / `TAP`(멤버 탭) / `DBL`(더블탭) / `HOT`(오브젝트 핫스팟). 상태 머신 정의는 기획서 §6.
**16:9 파노라마 팬**: 배경을 세로 100% 맞춤 → 가로 오버플로를 45초 주기로 좌↔우 왕복(ease-in-out). `reducedMotion` 시 중앙 고정.
**바닥 반사 씬**(08·09·12·13·17): 멤버 스프라이트를 수직 반전, 불투명도 18%, 발끝 기준 아래 접합, 높이 40%로 압축.

**해금 티어**: 시작(01 무료) / T1 각 80💧(02~06) / T2 각 150💧(07~14) / T3 각 250💧(15~20) / 21 = 20종 전수집 자동 보상. 총 3,100💧.

```ts
interface SceneSpec {
  id: `scene-${string}`;
  title: string;                 // 도감 표기명
  bg: string;                    // opt 매니페스트 키
  aspect: '9:16' | '16:9' | '1:1';
  tier: 0 | 1 | 2 | 3;           // 0=무료/보상
  price: number;                 // 💧 (tier0=0)
  special?: 'miniatures' | 'circle' | 'vpose';   // 특수룸(멤버 포즈 미등장)
  member?: {
    anchor: { x: number; y: number };  // 발/엉덩이 기준 %
    scale: number;
    idleA: PoseClip;                   // 포즈 + 보조 트랜스폼 + 주기
    idleB?: PoseClip & { every: number };
    reflection?: { opacity: number };  // 바닥 반사 씬
  };
  effects: EffectPlacement[];          // 종류·좌표·주기·클리핑 영역
  moodcuts: string[];                  // 컷 파일 키 (0개=특수룸)
  moodcutPos?: { x: number; y: number };
  tapWeights: { neutral: number; laugh: number; shy: number; surprised: number };
  dbl: ActionRef;                      // 더블탭 액션 id
  hotspots: { rect: Rect; action: ActionRef; label: string }[];
  hint: string;                        // 도감 힌트
  pan?: { speed: number };             // 16:9 파노라마
}
```
