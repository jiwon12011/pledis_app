## 6. 필수 보정 목록 (빌드 파이프라인 자동 처리)

1. `effect-water-droplets`: 근백색(RGB≥246 & 채도<8) → α=0 화이트키, 이후 연결요소 라벨링으로 6프레임 bbox 추출 → `opt/effects/droplet-f{1..6}.webp` + 매니페스트에 bbox 기록
2. `umbrella-color-mint`, `umbrella-color-purple`: 모서리 흰 사각 아티팩트 제거(캐노피 외접원 밖 α=0)
3. `C*-dance`, `C*-slide`: 외곽 8px 링 마스크(인접 프레임 잔상 제거)
4. `effect-clouds`: 2×2 슬라이스 → cloud-{1..4}
5. `effect-fireworks`: 가로 5등분 슬라이스 → firework-f{1..5}
6. `C*-walk-sheet`, `C*-run-sheet`: 무보정(181×181×8 정확) — 슬라이스 불필요, CSS steps용 원형 유지

## 7. 이미지 최적화 파이프라인 (`tools/optimize-assets.mjs`, sharp)

입력: `assets/generated/**` (원본 무수정) → 출력: `app/public/opt/**`

| 대상 | 처리 | 목표 |
|---|---|---|
| 배경 21장 | 최대변 1280px 리사이즈(kernel: nearest) → WebP q82 vs lossless(-z 9) 중 작은 쪽 | 장당 150~300KB, 총 43MB→약 5MB |
| 배경 썸네일 | 최대변 256px → WebP q75 | 도감 그리드용 |
| 배경 LQIP | 최대변 20px → base64 인라인(매니페스트에 포함) | 로딩 플레이스홀더 |
| 캐릭터·컷·이펙트·UI | 원치수 유지, §6 보정 후 WebP lossless | 픽셀 엣지 보존(손실 압축 금지) |
| 매니페스트 | `opt/manifest.json` — 파일별 {src, w, h, bytes, lqip?, frames?, bbox[]?} | 런타임 로더의 단일 진실 |

검증 게이트(파이프라인 끝에 자동): ① 출력 총량 < 12MB ② 배경 각 장 < 350KB ③ droplet 프레임 수 = 6 ④ 슬라이스 산출물 개수 일치. 실패 시 빌드 중단.

## 8. 런타임 로딩 정책

- 최초 로드: 현재 씬 배경 1장 + 선택 멤버 현재 포즈 + HUD 스프라이트 → 목표 초기 전송 < 800KB, LCP < 2.5s(4G)
- 프리로드: 해금 목록 기준 좌우 인접 씬 배경 2장(idle 시), 멤버 나머지 포즈(idle 시 순차)
- 미해금 씬: 썸네일만 로드(도감), 원본 로드 금지
- 캐시: (MVP 이후) 서비스워커 — 배경은 cache-first, 매니페스트는 stale-while-revalidate
